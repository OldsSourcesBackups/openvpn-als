Impossibilité de se connecter à un ordinateur Windows

Tout afficher

Cause : L'ordinateur Windows n'est pas configuré pour accepter des connexions à distance.

Solution : Activez le paramètre de bureau à distance sur l'ordinateur auquel vous voulez vous connecter.
Activation du paramètre de bureau à distance dans Windows Vista
Important

Pour modifier les paramètres de l'ordinateur distant, vous devez être administrateur sur l'ordinateur Windows.

  1. Sur l'ordinateur Windows, cliquez sur Démarrer , puis sur Panneau de configuration.

    Si vous êtes en Affichage classique, dans le volet de navigation, cliquez sur Page d’accueil du Panneau de configuration.

  2. Cliquez sur Système et maintenance, puis sur Système.

  3. Dans le volet de navigation, cliquez sur Paramètres d'utilisation à distance, puis, sous Bureau à Distance, cliquez sur l'option de votre choix.

    Si vous êtes invité à fournir un mot de passe administrateur ou une confirmation, tapez le mot de passe ou fournisseur la confirmation.

    Remarque L'authentification au niveau du réseau utilise des fonctionnalités de réseau améliorées de Windows Vista et est plus sécurisée que les deux autres options d'authentification.

  4. Cliquez sur Sélectionner des utilisateurs.

  5. Effectuez l'une des opérations suivantes :

    Pour

    Procédez comme suit

    Vérifier que vous avez accès

    Assurez-vous que votre compte d'utilisateur figure dans la zone. Si vous êtes administrateur sur l'ordinateur, vous y êtes automatiquement répertorié.

    Ajouter un compte utilisateur

    Cliquez sur Ajouter, puis, dans le champ, tapez le nom de compte d'utilisateur de votre choix.

Activation du paramètre de bureau à distance dans Windows XP
Important

Pour modifier les paramètres de l'ordinateur distant, vous devez être administrateur sur l'ordinateur Windows.

  1. Sur l'ordinateur Windows, cliquez sur Démarrer, puis sur Panneau de configuration.

    Si vous êtes en Affichage classique, dans le volet de navigation, cliquez sur Basculer vers l'affichage des catégories.

  2. Cliquez sur Performances et maintenance, puis sur Système.

  3. Sous l'onglet Distant, activez la case à cocher Autoriser les utilisateurs à se connecter à distance à cet ordinateur.

  4. Cliquez sur Choisir des utilisateurs distants.

  5. Effectuez l'une des opérations suivantes :

    Pour

    Procédez comme suit

    Vérifier que vous avez accès

    Assurez-vous que votre compte d'utilisateur figure dans la zone. Si vous êtes administrateur sur l'ordinateur, vous y êtes automatiquement répertorié.

    Ajouter un compte utilisateur

    Cliquez sur Ajouter, puis, dans le champ, tapez le nom de compte d'utilisateur de votre choix.

Solution : Assurez-vous que le pare-feu Windows autorise des exceptions pour le bureau à distance.
Autorisation d'exceptions via le pare-feu Windows sur un ordinateur exécutant Windows Vista
Important

Pour modifier les paramètres du pare-feu, vous devez être administrateur sur l'ordinateur Windows.

  1. Sur l'ordinateur Windows, cliquez sur Démarrer , puis sur Panneau de configuration.

    Si vous êtes en Affichage classique, dans le volet de navigation, cliquez sur Page d’accueil du Panneau de configuration.

  2. Cliquez sur Sécurité, puis, sous Pare-feu Windows, cliquez sur Autoriser un programme via le Pare-feu Windows.

    Si vous êtes invité à fournir un mot de passe administrateur ou une confirmation, tapez le mot de passe ou fournisseur la confirmation.

  3. Dans la liste Programme ou port, vérifiez que la case à cocher Bureau à distance est activée.

Autorisation d'exceptions via le pare-feu Windows sur un ordinateur exécutant Windows XP
Important

Pour modifier les paramètres du pare-feu, vous devez être administrateur sur l'ordinateur Windows.

  1. Sur l'ordinateur Windows, cliquez sur Démarrer, puis sur Panneau de configuration.

    Si vous êtes en Affichage classique, dans le volet de navigation, cliquez sur Basculer vers l'affichage des catégories.

  2. Cliquez sur Centre de sécurité, puis, sous Gérer les paramètres de sécurité pour, cliquez sur Pare-feu Windows.

  3. Assurez-vous que la case à cocher Ne pas autoriser d'exceptions est désactivée.

  4. Cliquez sur l'onglet Exceptions, puis vérifiez que la case à cocher Bureau à Distance est activée.

Cause : L'ordinateur Mac ne peut pas se connecter au réseau.

Solution : Assurez-vous que l'ordinateur Windows est activé et connecté au réseau.

Une connexion à distance peut échouer à cause de problèmes liés au réseau. Par exemple, une connexion à distance peut échouer si le réseau à distance est en panne ou si le câble Ethernet sur l'ordinateur Windows est débranché.

Cause : La stratégie réseau bloque les connexions.

Solution : Contactez l'administrateur réseau concernant la stratégie de sécurité de votre organisation.

Les stratégies de sécurité varient en fonction de l'organisation. Par exemple, si vous voulez vous connecter à un ordinateur Windows sur un réseau utilisant IPsec pour sécuriser le trafic réseau, vous devez avoir une exception de limite pour cet ordinateur Windows.

Cause : L'ordinateur Windows est introuvable.

Solution : Vérifiez que les paramètres réseau sur l'ordinateur Mac sont correctement configurés pour le type de connexion que vous utilisez.

Les paramètres de connexion Ethernet ou sans fil sur l'ordinateur Mac doivent être configurés pour le réseau auquel vous voulez vous connecter.

Solution : Vérifiez que vous avez entré le nom d'ordinateur correct.
Vérifiez le nom d'ordinateur dans Windows Vista

  1. Sur l'ordinateur Windows, cliquez sur Démarrer , puis sur Panneau de configuration.

    Si vous êtes en Affichage classique, dans le volet de navigation, cliquez sur Page d’accueil du Panneau de configuration.

  2. Cliquez sur Système et maintenance, puis sur Système.

  3. Sous Paramètres de nom d’ordinateur, de domaine et de groupe de travail, notez le nom de l'ordinateur ou son nom complet si l'ordinateur Windows est membres d'un domaine.

    Lorsque vous ouvrez Connexion Bureau à Distance sur l'ordinateur Mac, vous devez entrer ce nom.

Vérifiez le nom d'ordinateur dans Windows XP

  1. Sur l'ordinateur Windows, cliquez sur Démarrer, cliquez avec le bouton droit sur Poste de travail, puis cliquez sur Propriétés.

  2. Sous l'onglet Nom de l'ordinateur, notez le nom complet de l'ordinateur.

    Lorsque vous ouvrez Connexion Bureau à Distance sur l'ordinateur Mac, vous devez entrer ce nom.

Solution : Utilisez l'adresse IP de l'ordinateur si vous ne pouvez pas vous connecter à l'aide du nom d'ordinateur.

  1. Sur l'ordinateur Windows, cliquez sur Démarrer, sur Tous les programmes, sur Accessoires, puis sur Invite de commandes.

  2. À l'invite de commandes, tapez ipconfig, puis appuyez sur ENTRÉE.

  3. Sous Connexion au réseau local par carte Ethernet, indiquez l'adresse IP de l'ordinateur.

    Lorsque vous ouvrez Connexion Bureau à Distance sur l'ordinateur Mac, vous pouvez entrer l'adresse IP au lieu du nom d'ordinateur. Sur des ordinateurs exécutant Windows Vista, l'adresse IP est répertoriée comme adresse IPv4.

Solution : Assurez-vous que la connexion Internet sur l'ordinateur Mac fonctionne.

  • Ouvrez votre navigateur Web (par exemple, Safari), puis vérifiez que vous pouvez afficher une page Web.

Cause : L'identité de l'ordinateur Windows ne peut pas être vérifiée.

Solution : Vérifiez que la préférence de sécurité de Connexion Bureau à Distance est compatible avec la version de Windows en cours d'exécution sur l'ordinateur Windows.

Si un ordinateur Windows exécute Windows XP ou une version antérieure, ou Windows Server 2003 Service Pack 1 (SP1) ou une version antérieure, il ne peut pas fournir son identité pour vérification. Si l'ordinateur distant exécute une de ces versions de Windows et si la préférence de sécurité sur l'ordinateur Mac requiert une authentification pour se connecter, la connexion à distance échouera.

  1. Dans le menu Connexion Bureau à Distance, cliquez sur Préférences, puis sur Sécurité.

  2. Assurez-vous que l'option Ne pas se connecter en cas d'échec de l'authentification n'est pas activée.
