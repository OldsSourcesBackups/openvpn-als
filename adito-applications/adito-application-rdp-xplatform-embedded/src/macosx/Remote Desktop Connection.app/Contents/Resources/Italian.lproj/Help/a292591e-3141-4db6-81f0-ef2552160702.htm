Limitazione di una sessione a una singola applicazione Windows

Mostra tutto

Se si utilizza spesso un'applicazione specifica su un server che esegue Windows Server, è possibile risparmiare tempo avviando la sessione di desktop remoto solo con tale applicazione. Gli amministratori di rete possono, inoltre, limitare i computer client a una singola applicazione durante la connessione al server Windows. Il desktop di Windows non è visibile o disponibile quando viene selezionata questa preferenza.

  1. Dal menu Connessione desktop remoto scegliere Preferenze, quindi Applicazioni.

  2. Selezionare la casella di controllo.

  3. Nella casella Percorso e nome file dell'applicazione digitare il percorso e il nome file dell'applicazione che si desidera avviare.

    È, inoltre, possibile digitare il percorso della directory di lavoroCartella di installazione ed esecuzione di un'applicazione. La maggior parte delle applicazioni installate è progettata con una directory di lavoro fissa che non può essere modificata. Alcune installazioni precedenti, utilità e determinati script e file batch sono progettati per essere eseguiti in una cartella diversa da quella in cui si trova l'applicazione. per l'applicazione, se necessario.

Suggerimento Per chiudere la connessione, uscire dall'applicazione o disconnettersi dal computer Windows.
