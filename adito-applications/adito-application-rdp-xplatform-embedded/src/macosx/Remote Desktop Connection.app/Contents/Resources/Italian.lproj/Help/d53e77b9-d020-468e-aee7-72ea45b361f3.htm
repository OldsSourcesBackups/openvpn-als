Opzioni di autenticazione

Autenticazione computer remoto verifica che la connessione stabilita riguardi il computer Windows corretto. Sono disponibili tre opzioni di autenticazione.

  • Connetti sempre, anche se l'autenticazione non riesce

    Anche se in Connessione desktop remoto non è possibile verificare l'identità del computer Windows, la connessione viene stabilita ugualmente. Questa è l'opzione di autenticazione meno sicura.

    Nota Se su un computer Windows è in esecuzione Windows XP o versione precedente oppure Windows Server 2003 Service Pack 1 (SP1) o versione precedente, non sarà possibile ottenere l'identità per la verifica. Se si è a conoscenza del fatto che il computer Windows esegue uno di questi sistemi operativi precedenti, è possibile evitare gli avvisi di autenticazione scegliendo Connetti sempre, anche se l'autenticazione non riesce.

  • Avvisa se l'autenticazione non riesce (impostazione predefinita)

    Se in Connessione desktop remoto non è possibile verificare l'identità del computer Windows, viene visualizzato un messaggio di avviso in seguito al quale è possibile scegliere se continuare la connessione. Questa opzione di autenticazione migliora il livello di sicurezza rispetto all'opzione Connetti sempre, anche se l'autenticazione non riesce.

  • Avvisa se l'autenticazione non riesce

    Se in Connessione desktop remoto non è possibile verificare l'identità del computer Windows, non sarà possibile eseguire la connessione. Questa opzione di autenticazione utilizza funzioni di protezione migliorate quando si esegue la connessione a un computer su cui è in esecuzione Windows Vista. Inoltre, offre maggiore sicurezza rispetto alle altre due opzioni di autenticazione.

    Nota Se si seleziona questa opzione, non sarà possibile eseguire la connessione a computer che eseguono versioni del sistema operativo Windows precedenti a Windows Vista.
