コンピュータ間でコピーおよびペーストできない

すべて表示

原因 : クリップボードのデータ形式にはペーストできないものがあります。

対処方法 : そのクリップボードのデータ形式に対応しているプログラムに、いったんテキストをコピーします。
Windows ベースのコンピュータでコピーし、Mac にペーストする

  1. Windows ベースのコンピュータで目的のテキストをコピーします。

  2. Mac でテキストエディットを開き、テキストをペーストします。

  3. テキストエディットからテキストをコピーし、目的のファイルにペーストします。

Mac でコピーし、Windows ベースのコンピュータにペーストする

  1. Mac で目的のテキストをコピーします。

  2. Windows ベースのコンピュータでメモ帳またはワードパッドを開き、テキストをペーストします。

  3. メモ帳またはワードパッドからテキストをコピーし、目的のファイルにペーストします。

原因 : 目的のファイルが、Windows ベースのコンピュータからアクセスできるように設定されていません。

対処方法 : リモート デスクトップ セッションを開始する前に Windows ベースのコンピュータから Mac のディスクを利用できるようにしておきます。

  1. [RDC] メニューの [環境設定] をクリックします。

  2. [ドライブ] をクリックし、ポップアップ メニューで必要なオプションを選択します。

    ディスクまたはフォルダが利用可能になり、Windows ベースのコンピュータから Mac 上のファイルにアクセスできるようになります。

    接続後にこの変更を行った場合、選択したディスクまたはフォルダが利用できるのは次回の接続以降になります。

  3. [スタート] メニューの [コンピュータ] または [マイ コンピュータ] をクリックします。

    利用可能になった Mac のディスクまたはフォルダが、Windows ベースのコンピュータ上の他のすべてのディスクやフォルダと共に一覧に表示されます。

    メモ  [スタート] メニューが表示されない場合は、 [表示] メニューの [全画面表示] をクリックします。後でウインドウ モードに戻るには、画面の上端にポインタを置いて RDC のメニュー バーが表示されるのを待ち、メニュー バーが表示されたら、 [表示] メニューをクリックし、 [全画面表示] のチェックをオフにします。

  4. コピー元のディスクまたはフォルダを開きます。

  5. [スタート] メニューの [コンピュータ] または [マイ コンピュータ] をクリックします。

  6. コピー先のディスクまたはフォルダを開きます。

    これで、2 つのウインドウが開いて、コピー元の場所とコピー先の場所が表示されている状態になります。

  7. コピー元のドキュメントをコピー先の場所までドラッグします。

原因 : ターミナル サービスのグループ ポリシーでドライブの共有が許可されていません。

対処方法 : ターミナル サービスのグループ ポリシーの詳細については、ネットワーク管理者に問い合わせてください。

グループ ポリシーは、Remote Desktop Connection のどの環境設定よりも優先されます。 ターミナル サービスのグループ ポリシーでドライブの共有が禁止されている場合、コンピュータ間でファイルを共有することはできません。

原因 : ターミナル サービスのグループ ポリシーでクリップボードの共有が許可されていません。

対処方法 : ターミナル サービスのグループ ポリシーの詳細については、ネットワーク管理者に問い合わせてください。

グループ ポリシーは、Remote Desktop Connection のどの環境設定よりも優先されます。 ターミナル サービスのグループ ポリシーでクリップボードの共有が禁止されている場合、コンピュータ間でデータをコピーしてペーストすることはできません。
