セッションを 1 つの Windows ベースのプログラムに制限する

すべて表示

Windows Server を実行しているサーバーで、ある決まったプログラムを使用することが多い場合は、そのプログラムだけでリモート デスクトップ セッションを開始することで時間を節約できます。ネットワーク管理者が、Windows サーバーに接続中のクライアント コンピュータで使用できるプログラムを 1 つに制限することもできます。この環境設定が選択されている場合、Windows デスクトップは表示されないか、使用できなくなります。

  1. [RDC] メニューの [環境設定] をクリックし、 [プログラム] をクリックします。

  2. チェック ボックスをオンにします。

  3. [プログラムのパスとファイル名] ボックスに、起動するプログラムのパスとファイル名を入力します。

    必要に応じて、プログラムの作業ディレクトリプログラムがインストールされ、実行されるフォルダ。インストールされるプログラムのほとんどは、固定の作業ディレクトリを使用するようになっていて、作業ディレクトリを変更する必要はありません。古いプログラムや、ユーティリティ プログラム、スクリプトやバッチ ファイルの一部には、プログラムがあるフォルダとは別のフォルダで実行されるようになっているものもあります。のパスを指定することもできます。

ヒント 接続を終了するには、プログラムを終了するか、Windows ベースのコンピュータとの接続を切断します。
