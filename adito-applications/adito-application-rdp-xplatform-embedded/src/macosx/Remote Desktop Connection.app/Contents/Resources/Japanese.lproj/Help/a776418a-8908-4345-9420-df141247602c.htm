Windows ベースのコンピュータに接続できない

すべて表示

原因 : Windows ベースのコンピュータでリモート接続が許可されていません。

対処方法 : 接続先のコンピュータのリモート デスクトップ設定をオンにします。
Windows Vista のリモート デスクトップ設定をオンにする
重要

リモート デスクトップの設定を変更するには、Windows ベースのコンピュータの管理者権限が必要です。

  1. Windows ベースのコンピュータで、 [スタート] ボタン () をクリックし、 [コントロール パネル] をクリックします。

    クラシック表示の場合は、ナビゲーション ウインドウの [コントロール パネル ホーム] をクリックします。

  2. [システムとメンテナンス] をクリックし、 [システム] をクリックします。

  3. ナビゲーション ウインドウで [リモートの設定] をクリックし、 [リモート デスクトップ] で必要なオプションをクリックします。

    管理者のパスワードを求めるメッセージが表示されたらパスワードを入力し、その後の確認メッセージに従って、もう一度パスワードを入力します。

    メモ Windows Vista のネットワーク レベル認証では、セキュリティ機能が強化され、他の 2 つの認証オプションよりも安全性が向上しています。

  4. [ユーザーの選択] をクリックします。

  5. 必要に応じて、次の操作を行います。

    目的

    操作

    アクセス許可を確認する

    自分のユーザー アカウントがボックスの一覧に表示されていることを確認します。コンピュータの管理者のアカウントは自動的に一覧に表示されます。

    他のユーザー アカウントを追加する

    [追加] をクリックし、ボックス内に、追加するユーザー アカウント名を入力します。

Windows XP のリモート デスクトップ設定をオンにする
重要

リモート デスクトップの設定を変更するには、Windows ベースのコンピュータの管理者権限が必要です。

  1. Windows ベースのコンピュータで、 [スタート] ボタンをクリックし、 [コントロール パネル] をクリックします。

    クラシック表示の場合は、ナビゲーション ウインドウの [カテゴリの表示に切り替える] をクリックします。

  2. [パフォーマンスとメンテナンス] をクリックし、 [システム] をクリックします。

  3. [リモート] タブで、 [このコンピュータにユーザーがリモートで接続することを許可する] チェック ボックスをオンにします。

  4. [リモート ユーザーの選択] をクリックします。

  5. 必要に応じて、次の操作を行います。

    目的

    操作

    アクセス許可を確認する

    自分のユーザー アカウントがボックスの一覧に表示されていることを確認します。コンピュータの管理者のアカウントは自動的に一覧に表示されます。

対処方法 : Windows ファイアウォールでリモート デスクトップに対する例外が許可されていることを確認します。
Windows Vista を実行しているコンピュータで Windows ファイアウォールの例外を許可する
重要

ファイアウォールの設定を変更するには、Windows ベースのコンピュータの管理者権限が必要です。

  1. Windows ベースのコンピュータで、 [スタート] ボタン () をクリックし、 [コントロール パネル] をクリックします。

    クラシック表示の場合は、ナビゲーション ウインドウの [コントロール パネル ホーム] をクリックします。

  2. [セキュリティ] をクリックし、 [Windows ファイアウォール] で [Windows ファイアウォールによるプログラムの許可] をクリックします。

    管理者のパスワードを求めるメッセージが表示されたらパスワードを入力し、その後の確認メッセージに従って、もう一度パスワードを入力します。

  3. [プログラムまたはポート] ボックスの一覧で、 [リモート デスクトップ] チェック ボックスがオンになっていることを確認します。

Windows XP を実行しているコンピュータで Windows ファイアウォールの例外を許可する
重要

ファイアウォールの設定を変更するには、Windows ベースのコンピュータの管理者権限が必要です。

  1. Windows ベースのコンピュータで、 [スタート] ボタンをクリックし、 [コントロール パネル] をクリックします。

    クラシック表示の場合は、ナビゲーション ウインドウの [カテゴリの表示に切り替える] をクリックします。

  2. [セキュリティ センター] をクリックし、 [セキュリティの設定の管理] の [Windows ファイアウォール] をクリックします。

  3. [例外を許可しない] チェック ボックスがオフになっていることを確認します。

  4. [例外] タブをクリックし、 [リモート デスクトップ] チェック ボックスがオンになっていることを確認します。

原因 : Mac コンピュータがネットワークに接続できません。

対処方法 : Windows ベースのコンピュータの電源がオンで、ネットワークに接続されていることを確認します。

ネットワークの問題が原因で、リモート接続に失敗することがあります。たとえば、リモート ネットワークがダウンしている場合や、Windows ベースのコンピュータのイーサネット ケーブルが差し込まれていない場合には、リモート接続に失敗します。

原因 : ネットワーク ポリシーによって接続がブロックされています。

対処方法 : 組織のセキュリティ ポリシーについてネットワーク管理者に問い合わせてください。

セキュリティ ポリシーは組織によって異なります。たとえば、ネットワーク トラフィック保護のために IPsec を使用しているネットワークで Windows ベースのコンピュータに接続するには、その Windows ベースのコンピュータについて境界の例外が必要になります。

原因 : Windows ベースのコンピュータが見つかりません。

対処方法 : Mac のネットワーク設定が、使用している接続の種類に合わせて正しく設定されていることを確認します。

Mac のイーサネットまたはワイヤレス接続の設定は、接続先のネットワークに合わせて設定する必要があります。

対処方法 : 正しいコンピュータ名が入力されていることを確認します。
Windows Vista でコンピュータ名を確認する

  1. Windows ベースのコンピュータで、 [スタート] ボタン () をクリックし、 [コントロール パネル] をクリックします。

    クラシック表示の場合は、ナビゲーション ウインドウの [コントロール パネル ホーム] をクリックします。

  2. [システムとメンテナンス] をクリックし、 [システム] をクリックします。

  3. [コンピュータ名、ドメインおよびワークグループの設定] に表示されているコンピュータ名を控えておきます。Windows ベースのコンピュータがドメインのメンバの場合はフル コンピュータ名を控えておきます。

    この名前は、Mac で Remote Desktop Connection を起動したときに入力します。

Windows XP でコンピュータ名を確認する

  1. Windows ベースのコンピュータで、 [スタート] ボタンをクリックし、 [マイ コンピュータ] を右クリックして [プロパティ] をクリックします。

  2. [コンピュータ名] タブに表示されているフル コンピュータ名を控えておきます。

    この名前は、Mac で Remote Desktop Connection を起動したときに入力します。

対処方法 : コンピュータ名を使用して接続できない場合は、コンピュータの IP アドレスを使用します。

  1. Windows ベースのコンピュータで、 [スタート] 、 [すべてのプログラム] 、 [アクセサリ] 、 [コマンド プロンプト] の順にクリックします。

  2. コマンド プロンプトで「ipconfig」と入力し、Enter キーを押します。

  3. "Ethernet adapter ローカル エリア接続" の下に表示されるコンピュータの IP アドレスを控えておきます。

    Mac で Remote Desktop Connection を起動したときに、コンピュータ名の代わりに IP アドレスを入力できます。Windows Vista を実行しているコンピュータでは、IP アドレスは IPv4 アドレスとして表示されます。

対処方法 : Mac のインターネット接続が有効であることを確認します。

  • Web ブラウザ (Safari など) を開き、Web ページを表示できることを確認します。

原因 : Windows ベースのコンピュータの識別情報を確認できません。

対処方法 : Remote Desktop Connection のセキュリティの環境設定が Windows ベースのコンピュータで実行されている Windows のバージョンと互換性があることを確認します。

Windows ベースのコンピュータで、Windows XP 以前、または Windows Server 2003 Service Pack 1 (SP1) 以前のバージョンが実行されている場合、確認のための識別情報を取得することができません。 リモート コンピュータがこれらの Windows のバージョンのいずれかを実行していて、Mac のセキュリティ環境設定で接続時に認証を要求するように設定されている場合、リモート接続は失敗します。

  1. [RDC] メニューの [環境設定] をクリックし、 [セキュリティ] をクリックします。

  2. [認証が失敗した場合は接続しない] がオフになっていることを確認します。
