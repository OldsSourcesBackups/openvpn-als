認証オプション

リモート コンピュータの認証では、適切な Windows ベースのコンピュータに接続しているかどうかが確認されます。次の 3 つの認証オプションがあります。

  • [認証が失敗した場合も常に接続する]

    Remote Desktop Connection が Windows ベースのコンピュータの識別情報を確認できない場合でも、常に接続されます。これは安全性が最も低いオプションです。

    メモ Windows ベースのコンピュータで、Windows XP 以前、または Windows Server 2003 Service Pack 1 (SP1) 以前のバージョンが実行されている場合、確認のための識別情報を取得することができません。 Windows ベースのコンピュータでこれらの以前のオペレーティング システムのいずれかが実行されていることがわかっている場合は、 [認証が失敗した場合も常に接続する] をオンにして、認証についての警告が表示されないようにすることもできます。

  • [認証が失敗した場合は警告する] (既定の設定)

    Remote Desktop Connection が Windows ベースのコンピュータの識別情報を確認できない場合、警告が表示され、接続を続行するかどうかを選択できます。この認証オプションは、 [認証が失敗した場合も常に接続する] オプションよりも安全性が高い認証方法です。

  • [認証が失敗した場合は接続しない]

    Remote Desktop Connection が Windows ベースのコンピュータの識別情報を確認できない場合は接続できません。この認証オプションでは、Windows Vista を実行しているコンピュータに接続するときに、強化されたセキュリティ機能が使用されます。他の 2 つの認証オプションと比較すると、この認証オプションは最も安全性の高いものです。

    メモ このオプションを選択した場合、Windows Vista よりも前のバージョンの Windows オペレーティング システムが実行されているコンピュータに接続することはできません。
