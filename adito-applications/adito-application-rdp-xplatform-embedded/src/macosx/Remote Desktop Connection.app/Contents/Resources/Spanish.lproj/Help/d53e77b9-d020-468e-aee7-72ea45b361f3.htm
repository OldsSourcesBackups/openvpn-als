Opciones de autenticación

La autenticación del equipo remoto verifica que se está conectando al equipo basado en Windows correcto. Hay tres opciones de autenticación disponibles.

  • Conectar siempre, incluso si hay errores de autenticación

    Incluso si Conexión a Escritorio remoto no puede verificar la identidad del equipo basado en Windows, se conectará igualmente. Esta opción de autenticación es la menos segura.

    Nota Si el equipo basado en Windows ejecuta Windows XP Windows XP o anterior, o bien Windows Server 2003 Service Pack 1 (SP1) o anterior, no puede proporcionar su identidad para la verificación. Si tiene conocimiento de que el equipo basado en Windows está ejecutando uno de esos sistemas operativos anteriores, puede evitar las advertencias de autenticación al seleccionar Conectar siempre, incluso si hay errores de autenticación.

  • Avisarme si hay errores de autenticación (configuración predeterminada)

    Si Conexión a Escritorio remoto no puede verificar la identidad del equipo basado en Windows, recibirá una advertencia para que pueda elegir si desea continuar con la conexión. Esta opción de autenticación mejora la seguridad en comparación con la opción Conectar siempre, incluso si hay errores de autenticación.

  • No conectarse si hay errores de autenticación

    Si Conexión a Escritorio remoto no puede verificar la identidad del equipo basado en Windows, no podrá conectarse. Esta opción de autenticación utiliza funciones de seguridad mejoradas al conectarse a un equipo que ejecuta Windows Vista. Esta opción de autenticación mejora la seguridad en comparación con las otras dos opciones de autenticación.

    Nota Si selecciona esta opción, no podrá conectarse a equipos que ejecuten versiones del sistema operativo Windows que sean anteriores a Windows Vista.
